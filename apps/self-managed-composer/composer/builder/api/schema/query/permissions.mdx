    Returns a Relay Connection to a collection of Permission objects.

Access is permitted when any of the following condition(s) are met: 1. Always allowed.

## Input fields

| Argument  | Type                                                               | Description                                                                                      |
| --------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| `before`  | [`String`](/composer/builder/api/schema/scalar/String.mdx)                           | Specifies a cursor for use in combination with `last` to implement backward pagination.          |
| `after`   | [`String`](/composer/builder/api/schema/scalar/String.mdx)                           | Specifies a cursor for use in combination with `first` to implement forward pagination.          |
| `first`   | [`Int`](/composer/builder/api/schema/scalar/Int.mdx)                                 | A non-negative integer that specifies to return at most `first` edges after the `after` cursor.  |
| `last`    | [`Int`](/composer/builder/api/schema/scalar/Int.mdx)                                 | A non-negative integer that specifies to return at most `last` edges before the `before` cursor. |
| `offset`  | [`Int`](/composer/builder/api/schema/scalar/Int.mdx)                                 |                                                                                                  |
| `name`    | [`String`](/composer/builder/api/schema/scalar/String.mdx)                           | Filter for objects where name matches the specified value.                                       |
| `objType` | [`ID`](/composer/builder/api/schema/scalar/ID.mdx)                                   | Filter for objects where objType matches the specified ID.                                       |
| `orderBy` | [`PermissionOrder`](/composer/builder/api/schema/input_object/PermissionOrder.mdx)   | DEPRECATED. Prefer using sort_by instead as it allows ordering by many fields.                   |
| `sortBy`  | [`[PermissionOrder]`](/composer/builder/api/schema/input_object/PermissionOrder.mdx) | Allows specifying many field and direction pairs to sort results by.                             |

## Return fields ([PermissionConnection!](/composer/builder/api/schema/object/PermissionConnection.mdx))

###### <b>edges</b> ([[PermissionEdge]!](/composer/builder/api/schema/object/PermissionEdge.mdx))

List of edges containing the nodes in this connection.

###### <b>nodes</b> ([[Permission]!](/composer/builder/api/schema/object/Permission.mdx))

List of nodes in this connection.

###### <b>pageInfo</b> ([PageInfo!](/composer/builder/api/schema/object/PageInfo.mdx))

Pagination data for this connection.

###### <b>totalCount</b> ([Int!](/composer/builder/api/schema/scalar/Int.mdx))

Total count of nodes available.
