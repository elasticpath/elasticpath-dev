Represents a customer, which is an object that allows for logical
separation of Users, Instances, and other data that are specific to a
particular deployment of the Organization's product(s).

## Return fields

###### <b>allowAddAlertMonitor</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add an Alert Monitor to the Customer.

###### <b>allowAddComponent</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add a Component to the Customer.

###### <b>allowAddCredential</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add a Credential to the Customer.

###### <b>allowAddInstance</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add an Instance to the Customer.

###### <b>allowAddIntegration</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add an Integration to the Customer.

###### <b>allowAddUser</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can add a User to the Customer.

###### <b>allowConfigureCredentials</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User's Customer has access to legacy Credentials.

###### <b>allowEmbeddedDesigner</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether this Customer can use the Embedded Designer.

###### <b>allowEnableInstance</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether Instances may be enabled based on the utilization allowed by the current Plan.

###### <b>allowExecuteInstance</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether Instances may be executed based on the utilization allowed by the current Plan.

###### <b>allowManageAttachments</b> ([Boolean](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can manage Attachments related to this record.

###### <b>allowRemove</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can remove the Customer.

###### <b>allowUpdate</b> ([Boolean!](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can update the Customer.

###### <b>attachments</b> ([[Attachment]](/self-managed/composer/builder/api/schema/object/Attachment.mdx))

A JSON list of objects where each object has a key for name and URL that together describe the Attachment.

###### <b>avatarUrl</b> ([String](/self-managed/composer/builder/api/schema/scalar/String.mdx))

The URL for the avatar image.

###### <b>components</b> ([ComponentConnection!](/self-managed/composer/builder/api/schema/object/ComponentConnection.mdx))

The Customer the Component belongs to, if any. If this is NULL then the Component belongs to the Organization.

###### <b>createdAt</b> ([DateTime!](/self-managed/composer/builder/api/schema/scalar/DateTime.mdx))

The timestamp at which the object was created.

###### <b>credentials</b> ([CredentialConnection!](/self-managed/composer/builder/api/schema/object/CredentialConnection.mdx))

The Customer the Credential belongs to, if any. If NULL then Organization will be specified.

###### <b>description</b> ([String!](/self-managed/composer/builder/api/schema/scalar/String.mdx))

Additional notes about the Customer.

###### <b>externalId</b> ([String](/self-managed/composer/builder/api/schema/scalar/String.mdx))

Allows for mapping an external entity to a Builder record.

###### <b>id</b> ([ID!](/self-managed/composer/builder/api/schema/scalar/ID.mdx))

The ID of the object

###### <b>instances</b> ([InstanceConnection!](/self-managed/composer/builder/api/schema/object/InstanceConnection.mdx))

The Customer for which the Instance is deployed.

###### <b>integrations</b> ([IntegrationConnection!](/self-managed/composer/builder/api/schema/object/IntegrationConnection.mdx))

The Customer the Integration belongs to, if any. If this is NULL then the Integration belongs to the Organization.

###### <b>labels</b> ([[String!]](/self-managed/composer/builder/api/schema/scalar/String.mdx))

The labels that are associated with the object.

###### <b>name</b> ([String!](/self-managed/composer/builder/api/schema/scalar/String.mdx))

The name of the Customer, which must be unique within the scope of its Organization.

###### <b>org</b> ([Organization!](/self-managed/composer/builder/api/schema/object/Organization.mdx))

The Organization to which the Customer belongs.

###### <b>starred</b> ([Boolean](/self-managed/composer/builder/api/schema/scalar/Boolean.mdx))

Indicates whether the record is starred by the signed-in User.

###### <b>updatedAt</b> ([DateTime!](/self-managed/composer/builder/api/schema/scalar/DateTime.mdx))

The timestamp at which the object was most recently updated.

###### <b>users</b> ([UserConnection!](/self-managed/composer/builder/api/schema/object/UserConnection.mdx))

The Customer the user belongs to, if any. If this is NULL then Organization will be specified.
