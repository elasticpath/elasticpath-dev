    Creates a new AlertWebhook object.

Access is permitted when any of the following condition(s) are met: 1. The signed-in User has any of the following permissions for the associated Organization: [org_manage_instances].

## Input fields ([CreateAlertWebhookInput!](/self-managed/composer/builder/api/schema/input_object/CreateAlertWebhookInput.mdx))

| Argument           | Type                                      | Description                                                                           |
| ------------------ | ----------------------------------------- | ------------------------------------------------------------------------------------- |
| `name`             | [`String!`](/self-managed/composer/builder/api/schema/scalar/String.mdx) | The name of the AlertWebhook.                                                         |
| `url`              | [`String!`](/self-managed/composer/builder/api/schema/scalar/String.mdx) | The URL of the AlertWebhook.                                                          |
| `payloadTemplate`  | [`String!`](/self-managed/composer/builder/api/schema/scalar/String.mdx) | The template that is hydrated and then used as the body of the AlertWebhook request.  |
| `headers`          | [`String`](/self-managed/composer/builder/api/schema/scalar/String.mdx)  | A JSON string of key/value pairs that will be sent as headers in the Webhook request. |
| `clientMutationId` | [`String`](/self-managed/composer/builder/api/schema/scalar/String.mdx)  | A unique identifier for the client performing the mutation.                           |
