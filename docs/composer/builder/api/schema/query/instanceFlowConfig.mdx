    Returns the specified InstanceFlowConfig object.

Access is permitted when any of the following condition(s) are met: 1. The signed-in User has any of the following permissions for the associated Organization: [org_admin_instance_permissions, org_manage_instances, org_view_instances]. 2. The signed-in User has any of the following permissions for the object's 'instance' attribute: [instance_admin_permissions, instance_view, instance_edit, instance_remove]. 3. The signed-in User has any of the following permissions for the object's 'instance_Customer' attribute: [customer_admin_instance_permissions, customer_view_instances, customer_manage_marketplace_integrations]. 4. The signed-in User has any of the following permissions for the object's 'instance_Customer' attribute: [customer_access_marketplace_integrations]. 5. The signed-in User has any of the following permissions for the object's 'instance_Integration' attribute: [integration_admin_permissions, integration_view, integration_edit, integration_remove]. 6. The signed-in User has any of the following permissions for the object's 'instance_Integration_Customer' attribute: [customer_admin_manage_instances, customer_admin_integration_permissions, customer_manage_integrations].

## Input fields

| Argument | Type                              | Description                              |
| -------- | --------------------------------- | ---------------------------------------- |
| `id`     | [`ID!`](/docs/composer/builder/api/schema/scalar/ID.mdx) | The ID of the InstanceFlowConfig object. |

## Return fields ([InstanceFlowConfig](/docs/composer/builder/api/schema/object/InstanceFlowConfig.mdx))

###### <b>allowRemove</b> ([Boolean!](/docs/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can remove the InstanceFlowConfig.

###### <b>allowUpdate</b> ([Boolean!](/docs/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the signed-in User can update the InstanceFlowConfig.

###### <b>apiKeys</b> ([[String]](/docs/composer/builder/api/schema/scalar/String.mdx))

An optional collection of API Keys any of which, when specified, will be required as a header value in all requests to trigger execution of this IntegrationFlow for the associated Instance.

###### <b>executionResults</b> ([InstanceExecutionResultConnection!](/docs/composer/builder/api/schema/object/InstanceExecutionResultConnection.mdx))

The specific InstanceFlowConfig for the Instance being executed.

###### <b>flow</b> ([IntegrationFlow!](/docs/composer/builder/api/schema/object/IntegrationFlow.mdx))

The IntegrationFlow for which configuration is being specified for the associated Instance.

###### <b>id</b> ([ID!](/docs/composer/builder/api/schema/scalar/ID.mdx))

The ID of the object

###### <b>inFailedState</b> ([Boolean!](/docs/composer/builder/api/schema/scalar/Boolean.mdx))

Specifies whether the latest execution of this InstanceFlowConfig resulted in a failure.

###### <b>instance</b> ([Instance!](/docs/composer/builder/api/schema/object/Instance.mdx))

The configuration for the IntegrationFlow associated with the Instance.

###### <b>lastExecutedAt</b> ([DateTime](/docs/composer/builder/api/schema/scalar/DateTime.mdx))

The timestamp at which the InstanceFlowConfig was most recently executed.

###### <b>logs</b> ([LogConnection!](/docs/composer/builder/api/schema/object/LogConnection.mdx))

The IntegrationFlow which created the Log entry.

###### <b>monitors</b> ([AlertMonitorConnection!](/docs/composer/builder/api/schema/object/AlertMonitorConnection.mdx))

The IntegrationFlow that is being monitored by the AlertMonitor.

###### <b>testContentType</b> ([String](/docs/composer/builder/api/schema/scalar/String.mdx))

Content type of the payload for testing the IntegrationFlow associated with the Instance.

###### <b>testHeaders</b> ([JSONString](/docs/composer/builder/api/schema/scalar/JSONString.mdx))

Headers for testing this IntegrationFlow associated with the Instance.

###### <b>testPayload</b> ([String](/docs/composer/builder/api/schema/scalar/String.mdx))

Data payload for testing this IntegrationFlow associated with the Instance.

###### <b>webhookUrl</b> ([String!](/docs/composer/builder/api/schema/scalar/String.mdx))

The URL of the endpoint that triggers execution of the InstanceFlowConfig.
